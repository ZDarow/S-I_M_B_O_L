# Участие в разработке

Спасибо за интерес к проекту! Руководство по ремонту Renault Symbol —
открытый проект, и мы приветствуем любую помощь.

## Как помочь

- **Контент**: добавление новых разделов, исправление ошибок, дополнение существующих страниц
- **Код**: улучшение скриптов сборки, тесты, CI/CD
- **Дизайн**: темы, CSS, типографика
- **Перевод**: интернационализация (опционально)

## Процесс

1. Форкните репозиторий
2. Создайте ветку: `git checkout -b feature/my-change`
3. Внесите изменения
4. Запустите проверки: `make validate` + `make test`
5. Откройте Pull Request

## Правила

### Контент (Markdown)

- Файлы в `book/src/`, организованы по разделам
- Используйте ````markdown ```mermaid ```` для диаграмм
- Каждый раздел начинается с `#` (один уровень)
- Следуйте стилю `.markdownlint.yaml`
- Русский язык — официальный язык документации

### Код (Python)

- Python 3.10+ (используйте `str | None` вместо `Optional[str]`)
- PEP 8 (строка до 79 символов для кода, 72 для докстрингов)
- Все новые функции покрываются тестами
- `from scripts.mermaid_core import ...` для общей логики
- Избегайте внешних зависимостей (stdlib preferred)

### Коммиты

- Сообщения на русском
- Префиксы: `feat:`, `fix:`, `refactor:`, `chore:`, `docs:`, `test:`
- pre-commit hook проверяет: сборку, ссылки, TODO/FIXME

### Pull Requests

- Одно изменение — один PR
- Чек-лист в описании: что сделано, как тестировалось
- CI должен проходить (quality + build)

## Локальная разработка

```bash
# Установка зависимостей
make install-deps

# Установка pre-commit hook
make install-hooks

# Полная сборка
make build

# Локальный сервер
make serve

# Тесты
make test
```
